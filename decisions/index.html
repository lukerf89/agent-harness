02How to use

A worked decision record, not a quiz.

Each page is structured as an Architecture Decision Record: context (why it matters), research findings (3–4 paragraphs synthesizing the current state of the art), options (2–4 viable patterns with pros/cons), and a single recommendation with reasoning. Click any option card to lock in your choice; the selection persists in localStorage and updates the status of that question across all pages. Your decisions are local to this browser — there is no server, no tracking, no account.

The recommendations are deliberate but not authoritative. The agents synthesizing each page were briefed with the harness's four novel primitives (layered memory, proactive scheduler, source-provenance lineage, draft-as-versioned-artifact) and asked to pick options that compound with those primitives — so a recommendation is "this option fits the rest of the architecture best," not "this is objectively correct." If your context differs (you're building for a regulated industry, or you're not actually committed to all four primitives), the second-place options often become more attractive. The cards make the trade-offs explicit so you can override the pick with eyes open.

Use this directory as a working document. Take a pass, lock in decisions you're confident about, leave the others open. Revisit when a constraint changes — your selections persist until you clear them.
